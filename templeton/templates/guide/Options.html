1.1 Options ¶

‘pads -x’

Bash’s set -x flag

‘pads -S [-p|-b|-f|-d|-i|-h|-v] -[opt] [name...]’

Run automated system scripts and exit. The logic for this module is intended to interact with the OS itself. The options are as follows:

‘-p [-u|-l|-r|-i|-m|-h] [name...]’

Package Manager

‘-u [name...]’

Install the package(s) you want and exit. Just separate the package names with spaces, no commas or anything. If left blank PADS will update your machine.

‘-l’

List installed packages on machine

‘-r <name...>’

Remove supplied packages

‘-i’

Update info page

‘-m’

Update man page

‘-h’

Show help for package manager

‘-f [name]’

Show information about the directory you provide for name.

‘-d’

Show data about PADS.

‘-b’

Back up home directory and System to the repo. It will ask if you would like to back up to the server. Exits after execution.

‘-i’

Install, and set up mysql automatically

‘-h, --help’

Print help for the S module

‘-v, --version’

Print version for the S module

‘pads -P [-p:|-f:|-g:|-h|-v] [-m|-s|-d] name’

Run modules parsed in another language and exit. This module is intended to run full programs. Usually something other than Bash.

‘-p [-m|-s|-d] name’

For managing a Python Project or Module.

‘-f [-m|-s|-d] name’

For managing a Flask Project.

‘-g [-m|-s|-d] name’

For managing a Pygames Project or Module.

‘-h’

Show help for the P module

‘-v’

Show version for the P module

‘pads -R [-s|-h|-v] name’

Used to run any random script within the r_opts.sh file.

‘-s <name>’

Entry point to script

‘-h’

Show help for the R module

‘-v’

Show version for the R module

‘pads -T’

Used for test scripts in the ~/pads/src/T/ directory

‘pads -h’

Show help for PADS and exit

‘pads -v’

Show version for PADS and exit
